## 🧠 وكيل النمو الذكي (Growth Agent) — مشروع استراتيجي كبير (17 يوليو 2026)
> التموضع الجديد: Mawq3i ما بتبيع "متجر"، بتبيع نمو مُدار بالذكاء الاصطناعي. المواصفة الكاملة (الرؤية، مكونات النظام، قاعدة معايير السوق 2026، المخاطر) موجودة بـ `docs/growth-agent-spec.md`. هون بس نتابع تنفيذ المراحل.

**المرحلة 0 — الأساس (بنية البيانات، بدون أي ذكاء بعد):**
- [ ] جدول `product_daily_stats` (views/add_to_cart/purchases يومي لكل منتج)
- [ ] جدول `store_growth_events` (سجل كل قرار يتخذه الوكيل لاحقاً: نوع، سبب، تاريخ، حالة تنفيذ، نتيجة بعد أسبوع)
- [ ] جدول `market_benchmarks` (تخزين معايير السوق العالمية 2026 من الوثيقة، قابلة للتحديث الدوري)
- [ ] كارت/قسم بسيط بالـ Admin يعرض البيانات الخام هاي لأي متجر (تحقق قبل البناء فوقها)

**المرحلة 1 — التشخيص القائم على القواعد (بعد المرحلة 0):**
- [ ] Cron job يومي يحلل كل متجر بمنطق التشخيص (جدول القواعد بالوثيقة، قسم 3-B)
- [ ] صياغة التشخيص بلغة بشرية عربية عبر gpt-4.1-nano
- [ ] عرض التشخيصات بالـ Dashboard + خيار إرسال كإشعار WhatsApp

**المرحلة 2 — ربط Meta + TikTok Ads (بعد المرحلة 1):**
- [ ] Meta Marketing API integration
- [ ] TikTok Ads API integration
- [ ] ربط بيانات الإعلان بالـ orders الفعلية (مو بس نقرات)
- [ ] مقارنة أداء كل متجر بـ market_benchmarks

**المرحلة 3 — قرارات تلقائية محدودة (بعد المرحلة 2):**
- [ ] طبقة صلاحيات: تلقائي مقابل يحتاج موافقة (قسم 3-C بالوثيقة)
- [ ] واجهة موافقة بسيطة بالـ Dashboard
- [ ] فترة "بناء ثقة" إلزامية 30-60 يوم لكل متجر جديد (كل شي بموافقة بالبداية)

**المرحلة 4 — تعلم من بيانات Mawq3i نفسها (بعد ~3-6 أشهر تشغيل فعلي وتراكم بيانات):**
- [ ] تحليل شهري لأنماط عابرة للمتاجر
- [ ] استبدال market_benchmarks العامة تدريجياً بأرقام حقيقية من عملائنا

**المرحلة 5 — A/B Testing تلقائي (اختيارية، للمتاجر ذات الحركة الكافية فقط):**
- [ ] فلترة المتاجر الجاهزة إحصائياً
- [ ] بنية توليد نسخ بديلة + توزيع تلقائي للزوار

## 🥈 تحسينات جودة المنصة

- [x] **صلاحيات موظفين متعددة (Multi-user roles)** — ✅ **تم (12 يوليو 2026)**: جدول `store_staff` جديد بـ RLS، صفحة "الموظفون" لصاحب المتجر (إضافة/حذف موظف + تفعيل صلاحيات: الطلبات، المنتجات، الإحصائيات، العروض، الإعدادات)، الـ Sidebar والـ routing يتقيّدوا تلقائياً بصلاحيات الموظف. **ملاحظة مهمة**: لسا لازم ينشئ حساب دخول (Supabase auth) يدوياً لكل موظف بنفس الإيميل — مو تلقائي حالياً. Commit: `a1cbf0f`.
- [x] **تحليلات على مستوى المنتج (Product Analytics)** — ✅ **تم (12 يوليو 2026)**: جدول أداء لكل منتج (مشاهدات من GA4 + عدد الطلبات + الإيراد + نسبة التحويل)، مبني فوق بنية GA4 الموجودة أصلاً (per-product view tracking). Commit: `8d38532`.
- [x] **استوديو التسويق (Marketing Studio)** — ✅ **تم**: بديل عن فكرة المدونة (تاجر عادي ما بيعرف يكتب مقالات SEO). قسم واحد فيه 3 أدوات بالذكاء الاصطناعي: منشور سوشال ميديا (3 نسخ كابشن + هاشتاقات لأي منتج)، صورة ترويجية جاهزة للنشر (Instagram Story 1080×1920 مبنية على هوية المتجر)، ورسالة واتساب تسويقية جاهزة للصق بقائمة بث. Commit: `ca57446`.
- [ ] **حقول مخصصة (Custom Fields)** — مرونة إضافة معلومات خاصة لكل قطاع
